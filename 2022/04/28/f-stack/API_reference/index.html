F-Stack的API文档

发表于2022-04-28|更新于2022-04-28|dpdkf-stack
|字数总计:1.3k|阅读时长:5分钟|阅读量:

F-Stack（FF）是一个基于DPDK的高性能网络框架。

FF API 提供了标准的 Kqueue/Epoll 接口和微线程框架 （SPP）。

为了方便各种服务更简单、更快捷地使用F-Stack，F-Stack已经与Nginx和Redis集成。

头文件 ff_api.h 定义了以下 API，在使用 F-Stack 时，应使用该 API 替换系统调用。

初始化

ff_init函数

函数签名如下

该函数用于初始化F-Stack， 包括DPDK/FreeBSD网络栈等。

ff_run

函数签名如下

loop是一个回调函数，服务逻辑由用户实现，并由 F-Stack 的每次轮询调用。

Control API

ff_fcntl

函数签名如下

它对打开的文件描述符 fd 执行下面描述的操作之一。该操作由 cmd 确定。有关详细信息，请参阅 man fcntl。

ff_sysctl

ff_sysctl用于在运行时修改内核参数。但是，仅在启动 F-Stack 之前才支持它。

ff_ioctl

该函数操作特殊文件的基础设备参数。更多信息见 ioctl 手册。

网络API

ff_socket

ff_socket创建用于通信的终结点，并返回引用该终结点的文件描述符。更多信息见 socket手册。

ff_setsockopt & ff_getsockopt

这两个函数操作由文件描述符 sockfd 表示的套接字的选项。

ff_socketpair

该函数调用在指定类型的给定域中创建一对未命名的连接套接字，并使用可选的给定协议。

socket操作函数

socket操作函数。

ff_getpeername

该函数返回连接到套接字 sockfd 的对等方的地址，位于 addr 所指向的缓冲区中。

ff_getsockname

该函数返回套接字 sockfd 绑定到的当前地址，位于 addr 所指向的缓冲区中

ff_read & ff_readv

这两个函数尝试从 buf 开始读取文件描述符 fd 到缓冲区的字节数以计数。

ff_write & ff_writev

这两个函数最多写入从缓冲区指向 buf 到文件描述符 fd 所引用的文件的字节数。

ff_send & ff_sendto & ff_sendmsg

在套接字上发送消息的函数。

ff_recv & ff_recvfrom & ff_recvmsg

从套接字接收消息的函数。

ff_select

允许程序监视多个文件描述符，等待一个或多个文件描述符“就绪”用于某种类型的I/O操作(例如，可能的输入)。

ff_poll

ff_poll等待文件描述符上的事件。

Kqueue API

ff_kqueue

kqueue系统调用提供了一种通用方法，用于在事件发生或条件成立时通知用户，该方法基于称为过滤器的小段内核代码的结果。

Epoll API

ff_epoll_create

返回引用新 epoll 实例的文件描述符。

ff_epoll_ctl

此系统调用对文件描述符 epfd 引用的 epoll(7) 实例执行控制操作。

Micro Thread API micro_thread/mt_api.h

为了方便地开发无需复杂异步逻辑处理的异步程序(参考SPP的微线程框架)，F-Stack提供了一个微线程框架，以便使用异步调用实现同步编程。

UDP send/recv interface

使用随机套接字端口发送和接收 udp 数据包。

TCP send/recv interface

使用连接池发送和接收 tcp 数据包，保持活动状态默认值为 10 分钟。buf 的参数不能使用静态。

Dispatch API

数据包调度回调功能，由用户实现。

注册数据包 dispath 函数。

参数

  • data 此数据包的数据指针。
  • len 此数据包的长度。
  • queue_id 此数据包的当前队列。
  • nb_queues 要调度的队列数。

返回值

  • 0到nb_queues-1: 数据包将调度到的队列 ID。
  • FF_DISPATCH_ERROR (-1): 发生错误或数据包由用户处理，数据包将被释放。
  • FF_DISPATCH_RESPONSE (-2): 数据包由用户处理，数据包将得到响应。
